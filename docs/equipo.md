## Convenciones de contribución

1. Crear una rama desde `develop` con el formato `feature/<descripcion>`.
2. Escribir commits descriptivos en inglés o español (consistente por PR).
3. Asegurar que `mvn test` pasa antes de abrir un Pull Request.
4. Actualizar la documentación en `docs/` cuando se modifique comportamiento visible.
